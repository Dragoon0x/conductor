Design rules, not rectangles

CONDUCTOR doesn't translate your words into shapes. It translates them into design decisions. Every element it creates follows the rules a senior designer would apply instinctively.

1

Parse intent

Understands "pricing page" means comparison layout, feature tiers, CTA hierarchy.

2

Read context

Scans your existing Figma file for styles, components, tokens, and grid settings.

3

Apply rules

8px grid. Type scale ratios. Proper auto-layout constraints. Component reuse.

4

Generate

Creates Figma frames with proper structure, not absolute-positioned rectangles.

Design intelligence, built in

⊞ 8px grid

Every spacing value is a multiple of 8. Padding, margins, gaps. No arbitrary numbers.

◆ Type scale

Picks sizes from a ratio (1.2, 1.25, 1.333). Headings, body, captions all relate mathematically.

▤ Auto-layout

Every frame uses Figma auto-layout with proper padding, gap, and alignment. Zero absolute positioning.

◎ Component reuse

Reads your existing components library. Uses your buttons, cards, and inputs instead of creating new ones.

◧ Style tokens

References your defined color styles, text styles, and effect styles. Stays on-brand automatically.

◑ Hierarchy

Understands visual weight. Primary actions are prominent. Secondary content recedes. Information flows correctly.

Setup

// ~/.cursor/mcp.json
{
  "mcpServers": {
    "conductor": {
      "command": "npx",
      "args": ["-y", "conductor-figma"]
    }
  }
}

Install the CONDUCTOR plugin from the Figma Community. Open your Figma file, run the plugin, then chat:

"Create a pricing page with three tiers"

"Add a testimonial section using my existing card component"

"Convert this frame to auto-layout with 8px spacing"

"Audit this page — is my spacing consistent?"
